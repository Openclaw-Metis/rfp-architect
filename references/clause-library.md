# 台灣在地條款庫（可直接貼入 RFP）

這些是國際 RFP 範本通常沒有、但台灣委外案不可或缺的條款。write 模式視標的勾選貼入；review 模式檢查既有 RFP 是否遺漏。**請依個案調整等級、門檻與罰則數值。**

## A. 資通安全分級

> 本案系統依「資通系統防護需求分級原則」評估為【普／中／高】等級；得標廠商應落實對應等級之存取控制、加密、日誌稽核與弱點管理措施，並於各階段提供佐證。

- 分級越高，控制越嚴。涉及個資或關鍵業務者通常為中／高級。

## B. 廠商資安資格

> 得標廠商應簽署資訊安全承諾書與保密同意書，並提供 ISO 27001 驗證證書或完成資安自評查核表。

## C. 禁用產品與人員（台灣特有）

> 本案不得使用大陸廠牌之資通訊產品；廠商執行團隊成員不得為大陸地區人民。

- 公部門與關鍵基礎設施標準配備；遺漏可能於審標爭議或履約後衍生資安事件。

## D. 安全開發生命週期（SSDLC）

> 廠商應遵循安全軟體開發生命週期，於需求、設計、開發、測試、上線各階段執行安全檢核並留存紀錄。

## E. 特定標的附加要求

- **App**：取得行動應用基本資安檢測相關資安標章、符合無障礙設計。
- **網站**：導入 HTTPS。
- **涉個資**：資料落地（見 F）、個資保護措施。

## F. 資料落地與在地化

> 本案資料應儲存於中華民國境內；系統須支援繁體中文，包含台灣口音語音辨識、中英夾雜語句處理、符合台灣用語習慣之文字表達。

- **最常被直接翻譯國外範本忽略**，卻是能否在台灣上線的關鍵。

## G. 智慧財產權歸屬（提供可勾選選項，勿留空）

> 本案履約成果著作財產權歸屬：☐ 由機關取得　☐ 由廠商享有、機關取得授權利用。
> 著作人格權：☐ 廠商保留姓名表示權　☐ 其他約定＿＿。

- 原則：無特殊目的（國家機密 / 資安 / 高度客製）時，宜由廠商享有、機關取得授權（促進流通、降低成本）。需完整取得時於契約明確約定。
- **素材權利**：成果含音樂 / 圖片 / 影片片段時，須另取得公開傳輸 / 公開播送 / 公開上映授權；要求廠商驗收前提供授權書、著作財產權歸屬約定書（含其與員工之約定證明）。

## H. 驗收、保固與服務水準

> 廠商應於完成履約標的並通過驗收後請領款項；驗收付款前繳納保固保證金。保固期間【__】個月，涵蓋錯誤修正、功能調整與技術支援。
> SLA：系統可用率 ≥【99.9%】、API 回應時間 ≤【200ms】、未達標罰則【__】。

- 履約標的內容須詳載於契約及需求規格說明書；每條需求對應可驗收標準。

## I. 變更管理與變更請求流程（在 RFP 階段就先講清楚）

> 履約期間之需求變更應依正式變更請求（Change Request）流程辦理：提出者填具變更單，載明變更內容與原因、對範疇 / 時程 / 成本 / 風險之影響評估、不變更之後果與替代方案，經雙方變更控制窗口書面核准後始得執行；未經核准之變更不納入履約範圍。

- 變更請求分兩類：**範圍內變更**（客戶主導、原契約涵蓋）與**範圍外變更**（須另議時程 / 報價）。RFP 階段就揭露此流程，是抵禦範疇蔓延的核心機制。
- **為何放進 RFP**：委外失敗的首要根因是需求模糊與不受控的範疇蔓延；事前未約定變更流程，後期任何「再加一點」都會演變成延誤、超支與爭議。

## J. 退場、資料返還與移交（contract exit / transition-out）

> 契約屆滿、終止或解除時，廠商應於約定期限內：(1) 以雙方議定之開放格式完整返還機關所有資料與衍生資料，且不得保留或主張資料留置權；(2) 提供系統原始碼 / 設定 / 部署文件 / 維運手冊（依智財歸屬約定）；(3) 配合辦理知識移轉與服務移交，協助後手廠商或機關自行接管，確保服務不中斷。退場 / 移交之範圍、期限與配合義務應於契約明定。

- **為何放進 RFP**：缺退場與資料返還條款，機關易被廠商以資料 / 原始碼「綁架」，造成更換廠商困難、議價弱勢；CLM（契約生命週期管理）最佳實務要求把退場與資料返還條款前置設計。公部門與關鍵系統尤須具備。

## K. 專案治理與供應商盡職調查（把失敗預防框架寫成可投標條款）

> 機關側應指定產品負責人與技術窗口，並設能於【24】小時內裁決之專案發起人；履約期間採固定溝通節奏（每日 / 每週進度同步、共享專案管理工作區）。投標廠商應於建議書中：(1) 提出將**實際參與本案**之團隊名單、角色與資歷（非僅銷售或掛名人力）；(2) 視標的規模配合概念驗證（PoC）或啟動期付費技術評估，再進入完整開發。

- **為何放進 RFP**：常見失敗是「以精緻提案與低價得標，實際指派資格不足團隊」。要求揭露實際團隊、安排 PoC / 技術評估、明定甲方決策窗口與溝通儀式，能把治理從事後補救轉為投標即可檢核的條款。
