# 台灣政府採購法制（最有利標 / 評選 / 序位法）

委外 RFP 在台灣鑲嵌於《政府採購法》及其子法。write 模式據此設計招標 / 決標 / 評選章節；review 模式據此檢查程序合法性。**法條時有修正，引用前以全國法規資料庫最新版為準。**

> 本檔引用版本（查核日：2026-06-16）：《最有利標評選辦法》（民國 114 年 1 月 21 日修正，pcode A0030080）、《機關委託資訊服務廠商評選及計費辦法》（民國 114 年 5 月 23 日修正，pcode A0030078）、《採購評選委員會組織準則》（民國 115 年 5 月 8 日修正，pcode A0030103）。條號與數值請以全國法規資料庫最新公告為準；來源登錄見 `source-register.md`。

## 1. 招標方式三分法（差別＝廠商資格範圍）

- **公開招標** — 公告邀請所有不特定廠商；強調公開競爭。
- **選擇性招標** — 先資格審查建立合格廠商名單，再邀名單內投標。《政府採購法》第 20 條：適用經常性採購、審查耗時、需高額投標費用、資格複雜、研發等；經常性採購須建 6 家以上名單並定期檢討。
- **限制性招標** — 不經公開程序，直接邀特定廠商比價 / 議價。《政府採購法》第 22 條須符合法定條件（無廠商投標 / 獨家供應 / 緊急 / 後續擴充 / 小額採購等）。

## 2. 決標方式（決定 RFP 的評選邏輯）

- **最低標** — 符合規定且價格最低者得標。
- **評分及格最低標** — 先審查剔除表現太差者，合格者中價格最低者得標（達門檻即可，最終比價格）。
- **最有利標** — 不訂底價，綜合評選後由整體表現最好者得標（直接選「最好」）。**RFP / 建議書徵求最常綁定此方式。**

## 3. 異質採購與最有利標

- **異質採購**：標的在技術、品質、功能、效益或履約能力上顯著差異、無法僅以價格決標。認定依《政府採購法》第 52 條、施行細則第 66 條、《最有利標評選辦法》第 5 條。
- 面對異質採購 → 採最有利標，透過 RFP 廣納方案。
- **評選項目（八類）**：《最有利標評選辦法》§5 列八類評選項目供機關擇定——技術、品質、功能、管理、商業條款、過去履約績效、價格、財務計畫；CSR / 企業社會責任通常置於「管理」或「其他」項下。（實務常見的「技術 / 功能 / 管理 / 價格 / CSR」是此八類的簡化，設計配分時以八類為母體較準確。）
- **公開徵求建議書**：《政府採購法》第 34 條原則上招標前保密，但為公開徵求廠商提供參考資料則不在此限——機關得於招標前公開徵求對招標文件（規格 / 價格 / 履約條件）意見以了解市場、減少爭議；不得僅為省時直接向特定廠商詢價（洩密 / 圖利風險）。另有招標文件公開閱覽制度。
- **防綁標**：需求規格不得僅依單一現有廠商所提建議改寫（易生綁標疑慮）；一定規模以上專案宜先以 RFI（徵求資訊）或 RFC（徵求意見）蒐集多家廠商建議，再綜合製作需求規格。

## 4. 採購評選委員會（組成不合法 → 決標可被撤銷）

依《採購評選委員會組織準則》檢查委員會組成；這是獨立於《最有利標評選辦法》的 freshness source，更新時須同步查核 `source-register.md`。

- **委員 5 人以上**，具相關專門知識；**外聘專家學者不得少於 1/3**，且外聘委員不得為政府機關現職人員。
- 任務：招標前訂定 / 審定評選項目、評審標準、評定方式；辦理評選。**評選項目、子項、配分 / 權重須於招標文件明確載明，評選委員會不得變更或補充。**
- **召集人須機關首長，或由首長（或其授權人員）指定「一級主管以上」人員**；實務上「督學」「專門委員」非一級主管，不得任召集人。

## 5. 評定最有利標的三種方式（《最有利標評選辦法》§11）

機關須於招標文件載明採下列方式之一：

| 方式 | 法條 | 評定邏輯 | 適用情境 |
|---|---|---|---|
| 總評分法 | §12 | 價格納入評分者，以總評分最高、經首長或評選委員會過半數決定者得標；價格不納入者綜合考量總評分與價格 | 最直觀 |
| 評分單價法 | §13 | 價格不納入評分，以「價格 ÷ 總評分」**商數最低**者得標 | 想同時看品質與單位價格效益 |
| 序位法 | §15 | 各項評分加總轉序位，**序位合計值最低**者為序位第一 | 最常見；可消弭委員間打分差異 |

（條號以全國法規資料庫最新版為準。）

### 序位法詳述（最常用）

1. 個別委員對各廠商各評選項目評分後加總。
2. 依加總分數高低轉序位（最高＝序位 1）。
3. 彙整合計各廠商序位，**以序位合計值最低者為序位第一**。機關得另訂**合格（門檻）分數**（實務常見總平均 70 分），未達者不得作為最有利標——**70 分是機關自訂門檻、非法定固定值**，勿寫成通則。
4. 優點：消弭委員間打分差異（級距化，類比會考等級制）。
5. 價格納入評分 → 總評分最高且經首長或評選委員會過半數決定者為最有利標；價格不納入 → 綜合考量總評分與價格。
6. 同序位第一處理：再行綜合評選 / 擇配分最高項目得分合計較高 / 擇獲評定序位第一較多者。

## 6. 加權計分矩陣與價格、簡報的法定上下限

- 依重要性賦予各評選項目不同權重（例：技術 40%、價格 30%）。
- 組跨部門（IT / 工程 / 終端使用者）評選團隊，採一致的數值或加權計分法維持客觀。
- **價格權重的法定上下限**：採最有利標、且由廠商報價（價格納入評選）時，價格所占比率**不得低於 20%、不得逾 50%**（《最有利標評選辦法》§16 價格納入評分／§17 價格納入序位評比）。設計配分時務必落在此區間；任務關鍵系統可把價格壓到下限 20%、技術拉高。
- **固定價格給付的例外**：招標文件已訂明**固定價格給付**者，價格即使仍納入評選，其所占比率或權重**得低於 20%**（不受下限限制；依《最有利標作業手冊》）。因此「價格 <20%」只在固定價格案才合法，非固定價格案低於 20% 即違反 §16/§17。
- **簡報佔比上限**：廠商簡報及現場詢答列為評選項目者，配分或權重**不得逾 20%**，且簡報不得更改投標文件內容；廠商另提變更或補充資料者不納入評選。

### 兩套預設配分（依價格型態擇用，免每次重判價格門檻）

**情境 A：固定費用 / 預算已定案（固定價格給付）** —— 價格得低於 20%（例外成立）

| 面向 | 建議權重 |
|---|---:|
| 需求理解與解題架構 | 15% |
| 技術方案與系統架構 | 25% |
| 專案管理與交付能力 | 15% |
| 團隊與關鍵人員 | 15% |
| 資安與個資法遵 | 10% |
| 維運、保固與 SLA | 10% |
| 過去履約績效 | 5% |
| 價格合理性 | 5% |

**情境 B：廠商自報價格 / 非固定價格案** —— 價格須 ≥ 20%

| 面向 | 建議權重 |
|---|---:|
| 技術與需求理解 | 30% |
| 專案管理與交付 | 15% |
| 團隊能力 | 15% |
| 資安與個資 | 10% |
| 維運與 SLA | 10% |
| 價格 | 20% |

（兩表權重各合計 100%；情境 A 因屬固定價格給付，價格 5% 合法；情境 B 價格採 20% 下限。實際配分仍須依個案調整並落在法定區間。）

## 7. 程序時限與文件

- **等標期**（依招標期限標準、金額級距）：未達公告金額 ≥ 7 日；公告金額以上未達查核金額 ≥ 14 日；查核金額以上未達巨額 ≥ 21 日；巨額 ≥ 28 日。
- **投標須知**：規範投標規則、程序、資格、規格、契約條款（工程會有範本）。
- **釋疑**：廠商限期書面提疑義，機關須書面答覆並通知所有領標廠商，確保理解一致。

## 8. 資訊服務委外專法：《機關委託資訊服務廠商評選及計費辦法》

委外開發 / 資訊系統採購除適用《政府採購法》與《最有利標評選辦法》外，更直接受本辦法規範（依《政府採購法》第 22 條第 2 項授權訂定，pcode A0030078）。**這是資訊委外最對症的子法，write 設計章節、review 檢查合規均宜以此為底線。** 缺它不必然使採購無效（多數實質內容已被最有利標框架涵蓋），但補上能讓合規完整度與權威性顯著提升（高價值，定位 Major）。

### 8.1 資訊服務的法定範圍（§3）

提供與電腦軟硬體有關之服務，包括整體規劃、系統整合、系統稽核、系統管理、網路管理、軟體開發、軟體驗證、軟體維護、硬體維護、硬體操作、機房設施管理、備援服務、網路服務、顧問諮詢、資料庫建置、資料處理、資料登錄、訓練推廣等。→ 標的落在此清單內，本 skill 適用。

### 8.2 招標文件法定應載事項（§5）——對應 11 章節的法源底線

服務項目及工作範圍｜廠商應具備之電腦軟硬體 / 通信設施 / 資訊人力 / 經驗或實績等資格或資源及應檢附文件｜機關現有資訊作業環境｜工作時程｜材料設備成品規格｜驗收項目及標準｜計價及付款方式｜廠商應提建議書之內容（含主要工作項目時程、數量、價格、計畫內容、章節次序、頁數限制）｜智慧財產權之歸屬｜廠商於評選時須提簡報者其進行方式｜評審項目 / 標準 / 評選方式｜與評選優勝廠商議價及決標原則｜其他必要事項。→ review 時可逐款對照，作為「法定不可遺漏」底線。

### 8.3 資訊服務評審項目（§7）

廠商資源 / 資格｜專業技術能力｜品質保證能力｜計畫主持人及主要工作人員之經驗及能力｜資安及機密維護能力｜如期履約能力｜支援及維護能力｜價格｜教育訓練｜計畫執行方式｜建議書之完整性 / 可行性及對服務事項之瞭解程度｜其他。（採購軟體開發服務時，專業技術能力得包括在零成本或低成本前提下提供可自由存取 / 使用 / 修改 / 散布之共通性 API 開發或整合能力。）

### 8.4 評選與議價（準用最有利標）

- 資訊服務評選**準用最有利標**之評選規定；評選優勝廠商**得不以一家為限**。
- 優勝廠商一家者以議價辦理；二家以上者依優勝順序自最優者起依序議價。
- 計價與付款方式（§5 法定應載）對應「計費三法」，詳見 `clause-library.md` L 節。
