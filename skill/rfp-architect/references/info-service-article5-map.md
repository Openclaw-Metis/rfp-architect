# 資訊服務委外 §5 應載事項對照表（政府資訊服務案 review 用）

把《機關委託資訊服務廠商評選及計費辦法》（pcode A0030078）**第 5 條**法定應載事項，逐款對應到本 skill 的 11 章節骨架、`rfp_lint.py` check 與 review 嚴重度。**用途**：政府資訊服務案 review 模式可逐款比對，輸出比「缺某章節」更具法規可追溯性的 finding，例如：

```
§5 對照缺漏：
- 第 6 款 驗收項目及標準：缺漏（Blocker）
- 第 9 款 智慧財產權歸屬：不足（Blocker）—— 僅寫「另行約定」
- 第 3 款 機關現有資訊作業環境：缺漏（Major）
```

> 法源版本：民國 114 年 5 月 23 日修正（查核日 2026-06-16，見 `source-register.md`）。條號 / 應載款項以全國法規資料庫最新公告為準。缺 §5 款項多數不必然使採購無效（實質內容常已被最有利標框架涵蓋），但補齊能顯著提升合規完整度與權威性——定位為 Major，惟與驗收 / 智財 / 評選交集者升為 Blocker。

## 對照表

| §5 款項（資訊服務招標文件應載） | 對應 RFP 章節 | `rfp_lint.py` check | 缺漏 review 嚴重度 |
|---|---|---|---|
| (1) 服務項目及工作範圍 | 三 工作範疇（含 / 不含） | `scope` | Major（範疇未界定→範疇蔓延） |
| (2) 廠商應具備之軟硬體 / 通信 / 資訊人力 / 經驗實績等資格及應檢附文件 | 十一 投標須知（資格） | `submission` | Major |
| (3) 機關現有資訊作業環境 | 五 技術 / 非功能（整合架構、既有系統） | `nonfunc` | Major（廠商無從評估介接→報價失真） |
| (4) 工作時程 | 十 預算與時程里程碑 | `timeline` | Major |
| (5) 材料設備成品規格 | 四 功能需求 / 五 技術需求 | `functional` / `nonfunc` | Major |
| (6) 驗收項目及標準 | 十一 契約（驗收）＋各章驗收標準 | `acceptance` | **Blocker**（缺驗收＝需求未想清楚） |
| (7) 計價及付款方式 | 八 商業模式與定價（計費三法） | `pricing` | Major（§5 法定；缺法定計費法擇定） |
| (8) 廠商應提建議書之內容（主要工作項目時程 / 數量 / 價格 / 計畫內容 / 章節次序 / 頁數限制） | 十一 投標須知（建議書格式） | `submission` | Major |
| (9) 智慧財產權之歸屬 | 十一 契約（智財，含素材 / 分包鏈條） | `ip` | **Blocker**（含「另行約定」形同未約定） |
| (10) 廠商評選簡報之進行方式 | 九 評選（簡報列項者 ≤20%） | `evaluation` / `committee` | Minor（缺載方式）；簡報 >20% → Major（`presentation_weight` 規則） |
| (11) 評審項目、標準及評選方式 | 九 評選標準與配分 | `evaluation`（+ `committee` 合法性） | **Blocker**（未載配分 / 評選方式→決標爭議） |
| (12) 與評選優勝廠商議價及決標原則 | 九 評選 / 十一 契約（決標） | —（rubric 判定） | Major |
| (13) 其他必要事項 | 視個案（資安 / 個資 / 退場 / 雲端等） | `security` / `locale` / `exit` 等 | 視標的；涉個資 / 雲端 / 關鍵系統時 `exit`、`locale` 由 linter 升為 Blocker |

## §7 評審項目（評選配分設計時對照）

§7 列資訊服務評審項目：廠商資源 / 資格、專業技術能力、品質保證能力、計畫主持人及主要工作人員經驗能力、**資安及機密維護能力**、如期履約能力、支援及維護能力、價格、教育訓練、計畫執行方式、建議書完整性 / 可行性及對服務事項之瞭解程度、其他。

- write 設計第九章評選配分時，項目宜自此清單與《最有利標評選辦法》§5 八類評選項目交集取用；價格配分仍須落在 20%–50% 法定區間（固定價格給付案除外）。
- review 檢查評選項目是否涵蓋資安、計畫主持人 / 實際團隊（連動 `clause-library.md` K 治理盡職調查）。

## 使用提醒

- 本對照僅適用**政府資訊服務採購軌**；企業委外軌不套 §5，但 (1)(5)(6)(9) 等實質要件仍是好 RFP 的基本盤。
- §5 為「應載事項」清單，不等於品質判準——逐款「有寫」仍須以 `review-rubric.md` 判實質充分性。
